# Phase 2 — UI Design Contract

> Visual and interaction contract for the multi-plane medical image viewer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla JS + Canvas 2D) |
| Icon library | none (text labels only) |
| Font | system-ui, -apple-system, "Segoe UI", Roboto, sans-serif |

**Continuity:** Inherits all Phase 1 design tokens. Phase 2 extends the system with canvas-specific patterns (crosshair colors, orientation label styling, panel chrome) but does not alter the base tokens.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in Phase 2 |
|-------|-------|-------------------|
| xs | 4px | Gap between orientation label and canvas edge, crosshair line overshoot |
| sm | 8px | Slider track padding, A/C/S button internal padding, gap between panel chrome elements |
| md | 16px | Padding around viewer grid container, panel label bar internal padding |
| lg | 24px | Not used in Phase 2 |
| xl | 32px | Not used in Phase 2 |
| 2xl | 48px | Not used in Phase 2 |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- **Grid gap:** 2px between panels (not a spacing token; this is a structural divider that doubles as the panel border). Matches ITK-SNAP's thin black gaps between views.
- **Slider width:** 20px track width for slice navigation sliders. Not a spacing token; sized for comfortable drag interaction.

## Color

| Role | Value | Usage in Phase 2 |
|------|-------|-------------------|
| Dominant (60%) | #1e1e1e | App background behind viewer grid (visible as outer frame) |
| Secondary (30%) | #2d2d2d | Panel label bar background (thin bar above each canvas), slider track background, W/L preset button bar background |
| Accent (10%) | #4a9eff | Active A/C/S button highlight (indicates which view is in single-view mode), active W/L preset button highlight |
| Canvas background | #000000 | Canvas element background for all three image panels and the blank fourth panel |
| Text primary | #e0e0e0 | Orientation labels, slice index readout, preset button text |
| Text secondary | #a0a0a0 | Inactive A/C/S button text, panel identifier labels |
| Border | #3a3a3a | Panel label bar bottom border, slider track border |
| Surface hover | #363636 | W/L preset button hover, A/C/S button hover |
| Crosshair axial | #ffff00 (yellow) | Crosshair lines drawn on the axial panel |
| Crosshair coronal | #00ff00 (green) | Crosshair lines drawn on the coronal panel |
| Crosshair sagittal | #ff6600 (orange) | Crosshair lines drawn on the sagittal panel |
| Slider thumb | #4a9eff | Slider thumb/handle color (accent) |
| Slider track filled | #4a9eff at 40% opacity | Filled portion of slider track |
| Slider track empty | #3a3a3a | Unfilled portion of slider track |

**Accent reserved for:** Active A/C/S toggle button, active W/L preset button, slider thumb. Never applied to orientation labels, crosshair lines, or canvas backgrounds.

**Crosshair color rationale:** Three distinct high-saturation colors ensure each plane's crosshair is identifiable at a glance. Yellow, green, and orange are the conventional colors used by ITK-SNAP and 3D Slicer. These colors are deliberately NOT part of the UI accent palette -- they are medical imaging conventions.

**Canvas background rationale:** #000000 (pure black) is mandatory for medical image display. Grayscale pixel values of 0 must map to true black with no ambient light interference from a gray background. This matches ITK-SNAP and all clinical PACS viewers.

### Viewer Grid Container
- Fills all remaining space after sidebar (280px) and header (48px)
- CSS Grid: `grid-template-columns: 1fr 1fr; grid-template-rows: 1fr 1fr`
- Gap: 2px (structural divider, background #1e1e1e showing through)
- Background: #000000 (gap lines appear dark, almost invisible)

### Viewer Panel (one per view: axial, coronal, sagittal)

Each panel contains three layers stacked vertically:

#### Panel Label Bar (top strip)
- Height: 28px
- Background: #2d2d2d
- Border-bottom: 1px solid #3a3a3a
- Contents (left to right):
  - Panel name ("Axial" / "Coronal" / "Sagittal") at 12px semibold #a0a0a0, padding-left 8px
  - Slice readout ("128/256") at 12px semibold #e0e0e0, padding-left 8px
  - Spacer (flex: 1)
  - A/C/S button: 12px semibold, padding 4px 8px, background transparent, color #a0a0a0, hover background #363636, cursor pointer
  - In single-view mode: + button replaces A/C/S, same styling but with accent background #4a9eff and color #ffffff

#### Canvas Area (middle, fills remaining height)
- Background: #000000
- Canvas element centered within the area, sized by CSS for anisotropic correction
- `image-rendering: pixelated` on the canvas CSS (nearest-neighbor scaling)
- `ctx.imageSmoothingEnabled = false` in JavaScript

#### Orientation Labels (overlaid on canvas)
- Positioned absolutely within the canvas container
- 12px semibold #e0e0e0 with text-shadow `0 0 3px #000000` for legibility over bright images
- Padding: 4px from canvas edge
- Placement per view (after RAS+ normalization, radiological convention):
  - **Axial:** Left edge = "R", Right edge = "L", Top edge = "A", Bottom edge = "P"
  - **Coronal:** Left edge = "R", Right edge = "L", Top edge = "S", Bottom edge = "I"
  - **Sagittal:** Left edge = "A", Right edge = "P", Top edge = "S", Bottom edge = "I"

#### Crosshair Lines (drawn on canvas after image)
- Line width: 1px (canvas pixel, not CSS pixel)
- Compositing: drawn after the image data, using `ctx.strokeStyle` and `ctx.beginPath`/`ctx.lineTo`
- Each panel draws TWO crosshair lines intersecting at the current cursor position:
  - **Axial panel:** Vertical line at cursor X (full height), horizontal line at cursor Y (full width). Color: #ffff00
  - **Coronal panel:** Vertical line at cursor X (full height), horizontal line at cursor Z (full width). Color: #00ff00
  - **Sagittal panel:** Vertical line at cursor Y (full height), horizontal line at cursor Z (full width). Color: #ff6600
- Global alpha: 0.8 (slightly translucent so underlying anatomy remains visible)

#### Slice Navigation Slider (right edge of canvas area)
- Orientation: vertical, positioned along the right edge of the canvas container
- Width: 20px
- Track: background #3a3a3a, border-radius 4px
- Thumb: 12px diameter circle, #4a9eff, border 2px solid #e0e0e0
- Range: min=0, max=(dimension - 1), step=1
- Value display: shown in the panel label bar as "currentSlice/totalSlices"
- The slider is logically bound to the cursor axis for that view:
  - Axial slider controls cursor[2] (Z axis)
  - Coronal slider controls cursor[1] (Y axis)
  - Sagittal slider controls cursor[0] (X axis)
- Slider updates trigger re-render of ALL three panels (because crosshair positions change)

### Blank Panel (lower-right, VIEW-01)
- Background: #000000
- No canvas, no label bar, no slider
- In the future (Phase 3+), this panel may display a 3D preview or label panel. For Phase 2, it is empty.

### Single-View Mode (VIEW-04)

When user clicks A, C, or S button:
- Three panels are hidden (`display: none`)
- Active panel expands to fill entire grid (`grid-column: 1 / -1; grid-row: 1 / -1`)
- The A/C/S button in the label bar is replaced by a "+" button
- Canvas is resized via ResizeObserver to fill the new container size
- Slider remains on right edge, now taller

When user clicks "+" button:
- All four grid cells are restored
- All canvases are resized
- A/C/S buttons return to each panel

### W/L Preset Buttons (in left sidebar, Phase 1 tool panel area)

The Phase 1 sidebar at 280px width hosts the W/L presets when a CT volume is loaded.

- Section heading: "Window Presets" at 12px semibold #a0a0a0, padding 16px 16px 8px 16px
- Four buttons stacked vertically:
  - "Brain" (W:80, L:40)
  - "Bone" (W:3000, L:500)
  - "Lung" (W:1000, L:-500)
  - "Abd" (W:450, L:125)
- Button styling:
  - Width: 100% minus 32px (16px padding each side)
  - Margin: 0 16px 8px 16px
  - Padding: 8px 16px
  - Background: #2d2d2d
  - Border: 1px solid #3a3a3a
  - Border-radius: 6px
  - Font: 14px regular #e0e0e0
  - Hover: background #363636
  - Active (current preset): background #4a9eff, color #ffffff, border-color #4a9eff
  - Cursor: pointer
- **Visibility:** Shown only when `metadata.modality === "CT"` (WLVL-04). Hidden entirely for MR.

### Current W/L Display (in left sidebar, below presets or at top if MR)

- Two-line readout:
  - "W: {windowWidth}" at 12px regular #a0a0a0
  - "L: {windowCenter}" at 12px regular #a0a0a0
- Padding: 16px
- Updates in real time during ctrl+drag

## Interaction Contract

### Slice Navigation (VIEW-03, VIEW-07)
- **Trigger:** Drag slider OR click on canvas
- **Slider drag:** Updates the cursor axis for that panel, triggers re-render of all three panels (crosshair sync)
- **Canvas click:** Converts canvas pixel coordinates to voxel coordinates using inverse CSS scaling, updates the two NON-slice axes in the shared cursor, triggers re-render of all panels
- **Latency target:** <16ms per re-render (60fps). Axial is zero-copy (subarray). Coronal/sagittal require strided copy but should complete in <5ms for 512x512.

### Crosshair Synchronization (VIEW-07)
- **Trigger:** Click (not drag) on any canvas panel
- **Response:** Update shared ViewerState cursor[x,y,z]. All three panels re-render with new crosshair lines and potentially new slice (if the click changed the slice axis for another panel).
- **Visual feedback:** Crosshair lines immediately snap to clicked position. No animation.
- **Mouse cursor:** Default cursor (arrow) over canvas at rest. During Ctrl+drag, cursor changes to `cursor: grab`.

### Window/Level Adjustment (WLVL-02)
- **Trigger:** Ctrl+mousedown then drag on any canvas
- **Horizontal drag right:** Increase window width (more contrast range)
- **Horizontal drag left:** Decrease window width (less contrast range)
- **Vertical drag up:** Decrease window center (brighter)
- **Vertical drag down:** Increase window center (darker)
- **Sensitivity:** Proportional to current window width divided by 300. This makes sensitivity scale with the current range.
- **Minimum window width:** 1 (never zero or negative)
- **Response:** Rebuild LUT (or recompute per-pixel for float32), re-render all three panels
- **Latency target:** <16ms total (LUT rebuild + 3 panel renders)
- **Conflict avoidance:** Ctrl+scroll is NOT intercepted (remains browser zoom). Only Ctrl+drag is captured. Check both `e.ctrlKey` and `e.metaKey` for macOS compatibility. Call `e.preventDefault()` on Ctrl+mousedown to prevent text selection.

### Auto Window/Level (WLVL-01)
- **Trigger:** Automatic on volume open
- **Response:** Server provides `window_center` and `window_width` in volume metadata (5th-95th percentile, excluding zero-padding). Client sets initial W/L from these values.
- **No user interaction required.**

### W/L Preset Selection (WLVL-03)
- **Trigger:** Click preset button (Brain/Bone/Lung/Abd)
- **Response:** Set window width and center to preset values. Highlight active preset button. Re-render all panels.
- **Deactivation:** When user performs a Ctrl+drag adjustment after selecting a preset, the preset button deactivates (returns to default styling). The custom W/L values persist.

### Single-View Toggle (VIEW-04)
- **Trigger:** Click A/C/S button on any panel's label bar
- **Response:** Transition to single-view mode showing that panel full-size. Button label changes to "+".
- **Return trigger:** Click "+" button
- **Response:** Restore 4-panel layout. Canvas sizes recalculated via ResizeObserver.
- **Latency target:** <100ms including canvas resize

### Initial Load (VIEW-02)
- **Trigger:** Volume data received from server (continuing from Phase 1 "Open Volume")
- **Response:**
  1. Parse metadata (dimensions, spacing, modality, auto-window values)
  2. Store Float32Array volume in module-level variable
  3. Initialize ViewerState with cursor at center: [dimX/2, dimY/2, dimZ/2]
  4. Set W/L from server-provided auto-window metadata
  5. Render all three panels
  6. Show/hide W/L presets based on modality
- **Transition:** Phase 1 browser UI (volume list + detail panel) remains visible in the sidebar. The detail panel area transforms into the viewer grid.

## Responsive Behavior

Not applicable. Desktop tool for researchers/radiologists (REQUIREMENTS.md explicitly excludes mobile/tablet). Minimum viewport: 1024x768. Viewer grid fills available space via CSS Grid `1fr` units. Canvas elements resize via ResizeObserver when the container changes. No responsive breakpoints.

## Accessibility Baseline

- Slice sliders use semantic `<input type="range">` with `aria-label="Axial slice"` (and equivalent for coronal/sagittal), `aria-valuemin="0"`, `aria-valuemax="{dim-1}"`, `aria-valuenow="{current}"`
- A/C/S/+ buttons use semantic `<button>` elements with `aria-label="Toggle axial single view"` (and equivalent)
- W/L preset buttons use semantic `<button>` elements with `aria-pressed="true/false"`
- Canvas elements have `role="img"` and `aria-label="Axial view of loaded volume"` (and equivalent)
- Ctrl+drag W/L interaction has no keyboard equivalent in Phase 2 (the preset buttons provide keyboard-accessible W/L control for CT; MR users can rely on auto-windowing)
- Color contrast: #e0e0e0 on #2d2d2d = 9.7:1 (exceeds WCAG AAA). Orientation labels use text-shadow for legibility on variable backgrounds.
- Focus ring: 2px solid #4a9eff with 2px offset on all interactive elements (inherited from Phase 1)

## Phase-Forward Notes

These are NOT in scope for Phase 2 but inform design decisions made here:

- **Phase 3** adds segmentation overlay on each canvas. The render pipeline in ViewerPanel must be compositable: render base image, then overlay segmentation with per-label colors and transparency. Canvas compositing (`globalAlpha`, `globalCompositeOperation`) or a second overlay canvas per panel.
- **Phase 3** adds a label dropdown and transparency slider in the left sidebar. The sidebar layout should accommodate additional controls below the W/L presets.
- **Phase 4** adds a tool panel (EDIT-10: "tool panel on left side with light gray background"). Per Phase 1 forward-note, recommend using #3a3a3a (lighter dark) rather than literal light gray, to maintain dark theme consistency. The 280px sidebar width should accommodate tool panel controls.
- **Crosshair lines** will need to be drawn ABOVE the segmentation overlay in Phase 3. Render order: base image, then segmentation overlay, then crosshair lines.
